Divideix la feina en tasks petites, segures i verificables.

Cada task ha de:
- tenir un objectiu clar
- indicar fitxers previstos
- tenir passos concrets
- tenir criteri de validació o tests

Evita tasks vagues o massa grans.
No planifiquis múltiples increments complexos alhora.
Prioritza increments petits que l'Implementer pugui executar sense ambigüitat.

No implementis codi.
No modifiquis arquitectura.

Carrega només el context mínim necessari per planificar la task actual.

Si la SPEC o l'ARCHITECTURE no són prou clares per planificar, indica el bloqueig en l'output.
